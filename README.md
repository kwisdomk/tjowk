## Content Model

The site is data-driven. Public content lives mainly in:

- `content/profile.json`
- `content/status.json`
- `content/certs.json`
- `content/timeline.json`
- `content/projects/*.json`
- `content/journal/*.md`

Content is loaded through `lib/content/` and validated with Zod schemas.
